# openclaw-control-console-skill

一份用于 OpenClaw / Agent 工作区的控制台化 Skill。

它的目标是把主窗口变成**指挥台**，把重任务移到后台或独立会话，并且把开工卡、进度回报、超时规则、多任务并行边界统一下来。

## 这份 Skill 解决什么问题

- 主窗口越来越重，token 占用不断膨胀
- 长日志、长工具输出、大段证据被直接塞进聊天
- 重任务切后台后长期不回报，让人分不清是在跑、卡住还是坏掉
- 多任务同时推进时，容易互相串味或写同一资源打架

## 核心规则

- **主窗口轻量化**：主窗口只负责指令、状态、决策、收口
- **重任务后台化**：重活放到后台、独立会话或项目线程
- **短开工卡**：后台重任务开始前先发 task_id / 目标 / 当前步骤 / 下一次回报时间 / 可用追问词
- **固定回报节奏**：有新证据 5 分钟内回；无新证据最晚 10 分钟也要回；超过 15 分钟未回视为异常
- **固定追问词**：`进度`、`进度 task_id`、`当前卡点`、`停止当前任务`
- **并行边界**：允许多任务并行，但同项目 / 同 repo / 同服务 / 同数据库 / 同部署目标，只允许 1 个写入型重任务

## 适用场景

适合以下需求：

- 想让 Agent 主窗口更短、更稳、更像控制台
- 想把重任务拆出去，避免污染主线程上下文
- 想让后台任务必须定时回报，避免“静默跑飞”
- 想让多任务并行时减少互相干扰
